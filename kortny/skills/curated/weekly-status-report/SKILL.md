## Goal

Compile a status report that reads like a sharp chief-of-staff wrote it.

## Steps

1. Gather the week's raw material: channel history, completed tasks, shipped artifacts, prior report if one exists.
2. Group by workstream, not by day. A workstream with no movement gets one line: "No change."
3. For each workstream: what shipped/moved, what's blocked (and on whom), what's next.
4. Separate facts from interpretation. Metrics get exact numbers with week-over-week deltas where available.
5. Close with "Risks & asks" — things leadership can actually act on.

## Rules

- No filler praise ("great progress this week!").
- Blockers name the blocking party or dependency precisely.
- Keep it scannable: bold workstream names, one-line bullets, under 300 words for a typical team week.
