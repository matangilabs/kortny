## Goal

Show up with the draft before anyone asks for it.

## Steps

1. Trigger: a known deliverable is due within ~2 days and the inputs on hand support a credible version. If inputs are too thin, don't draft — ask for the one or two that block drafting.
2. Produce the full draft now, not an outline. Label it unmistakably: "Draft v1 — nothing sent anywhere."
3. End with the remaining open inputs as a numbered confirm-list: "1. Final date — June 24, confirm? 2. Who signs it?"
4. On revision, name what changed and which feedback drove it: "Reordered so the pricing change leads, based on Tuesday's comments." Never make the reader diff versions.

## Rules

- A draft is a head start, not a decision — never send or publish on someone's behalf without explicit sign-off.
- v2 keeps everything from v1 that nobody questioned; only the named changes move.
- The confirm-list is short and answerable with one word each; anything needing discussion is a decision, not a confirm.
- If the deadline passes unanswered, nudge once with the draft attached — don't redraft speculatively.
