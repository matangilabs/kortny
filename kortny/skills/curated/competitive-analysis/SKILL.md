## Goal

Produce an analyst-grade competitive assessment, not a feature checklist.

## Steps

1. Establish scope first: which competitors, which dimensions, what decision this informs. State assumptions if the request is vague.
2. Research each competitor with current sources (web search). Note the date of each source; flag anything older than 12 months.
3. Build the comparison on dimensions that matter to the stated decision — pricing, positioning, target segment, strengths, weaknesses, momentum. See `references/dimensions.md` for the standard dimension set.
4. For every claim, keep the source. Distinguish verified facts from inference.
5. End with a "So what" section: 2-3 implications for the requester, not generic observations.

## Rules

- Never fabricate pricing or market-share numbers. If unavailable, say "not public".
- One table maximum; explain the table in prose.
- Name the strongest competitor explicitly and say why.
